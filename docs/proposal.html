Detecting Embargo Breach in a Multi-Agent AI Communication Network

Author

CSM TraceForce

Published

June 10, 2026

1 Motivation

Companies are beginning to hand communication and decision work to teams of AI agents that message one another across shared channels. The speed and volume that makes these agents useful also makes misconduct hard to spot.

VAST Challenge 2026 Mini-Challenge 1 (MC1) presents this problem in concrete form. A property technology company called TenantThread runs seven AI agents during a public relations response while a merger sits under embargo. A compliance agent called the Judge is meant to watch for violations. Despite this, embargoed merger information reaches the outside world. The question is whether an agent bypassed the rules or the monitoring itself broke down.

A visual analytics tool can surface the abnormal behaviour and trace it back to the messages that prove it.

2 Objectives

This is an initial design. The signals and views described here are likely to evolve as the data is explored and the application is built.

The project aims to build an interactive Shiny application that lets a non-technical compliance analyst investigate the TenantThread scenario and reach an evidence-based judgement on the breach. Four objectives guide the design.

  1. Define normal and surface what departs from it. The analyst sets the boundary between the normal period and the suspect period. The application would then show which agents drift from their own habitual channel use, and by how much, once the suspect period begins.

  2. Detect communications that reached outside without oversight. The application would flag public posts an agent made without leaving a trace on a monitored channel. The analyst would see which posts bypassed the Judge, who made them and when they first appeared.

  3. Show how the structure of communication changed. The application would render the agent-to-agent network and the agent-to-channel network round by round, with a difference view marking links that were new, dropped or unchanged. The analyst would see whether any agent was cut out of the conversation and how the network reorganised around the breach.

  4. Connect every signal back to the message text. The analyst would open any flagged message to read its content alongside the agent’s private reasoning. This closes the path from an aggregate pattern to the specific evidence behind it.

3 Data

The data comes from VAST Challenge 2026 Mini-Challenge 1. The source is a nested JSON file holding 23 timeline rounds. Each round records an event headline, a short narrative and the messages exchanged in that round. The data covers seven AI agents communicating over six channels of decreasing accountability.

The raw structure would be flattened into three working tables. A message table of roughly 912 rows holds the sender, channel, recipients, content and the agent’s private reasoning fields. A round table of 23 rows holds each round’s narrative. A network table holds the directed links between agents derived from the recipient lists.

A legitimate announcement should travel from the most accountable channel down to the least, starting as internal discussion and ending as an attributed public post. The breach is content reaching an unmonitored public channel without that accountable path. The table below sets out the six channels and which the Judge can see.

Rank Channel Layer Monitored
1 Communications huddle Internal Yes
2 One-on-one chat Internal Yes
3 Side huddle Internal No
4 Official post Public No
5 Personal post Public No
6 Anonymous post Public No

4.1 Diagram

The methodology runs as a three-stage pipeline. Raw data is prepared, behavioural signals are computed against a baseline the analyst chooses and the signals are shown through four interactive sections.

The methodology unfolds in three phases. Each phase produces what the next one needs.

  • Phase 1 - Data Preparation
  • Phase 2 - Analytical Computation
  • Phase 3 - Interactive Presentation

The work begins by getting the data into shape. The nested timeline is flattened into three tables (messages, rounds and network links). Two reference structures are added on top. A channel hierarchy ranks each channel and marks which the Judge can see, and a topic vocabulary tags each message by keyword. With these in place, the analysis can begin.

The next phase builds three behavioural signals against the chosen baseline. Abnormality compares each agent’s channel use to its normal pattern. Bypass flags public posts an agent made without leaving a trace on a monitored channel. Network difference compares the communication structure before and after the baseline. The output is a set of signals ready to be shown.

The final phase builds the Shiny application around the signals. The four sections follow an investigation. The analyst first defines what is normal, then detects the bypass, then reads the network change and finally inspects the message evidence. The layout is described under Prototype Sketches.

5 Prototype Sketches

The application uses a four-tab layout with Activity as the entry point. Controls sit in a left sidebar, the visualisation fills the main panel. These layouts are a first pass and will change during development.

The baseline slider on the Activity tab is the one control that affects every tab. Move it, and the abnormality, bypass and network views all recompute against the new baseline. Every other input affects only its own tab.

  • The bypass
  • Network
  • Topics and evidence

Sets the baseline and shows where each agent is using channels it does not normally use. The main view is an abnormality grid placing agents against channels, with each cell shaded by how unusual that channel is for that agent. A round slider with a play control steps through time. A second number reports how many flagged messages survive under an earlier baseline, indicating whether the finding is robust.

Finds public posts that reached outside without oversight. Posts get flagged as a candidate when the agent made no monitored-channel message in the same round, and graded stronger or likely explained depending on whether the agent left any monitored trace in the rounds before. The flagged posts appear as a time series above a table of the actual content.

Shows how the communication structure changed. The graph view plots directed agent-to-agent links on a circular layout. The bipartite view shows which channels each agent used. The difference view colours each link by whether it was new, dropped or unchanged between the baseline and suspect periods.

Connects the signals to the actual text. Explore by round to see what each agent talked about, or by topic to see every round and channel where it appeared. An evidence table beneath the chart exposes each message’s content and the agent’s private reasoning.
