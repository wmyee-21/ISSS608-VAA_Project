Key context

The scenario is VAST Challenge 2026 Mini-Challenge 1. A property-technology company named TenantThread runs seven AI agents through a public-relations crisis while a merger with CivicLoom sits under embargo. A compliance agent named the Judge watches a subset of channels for violations. The analytical question is whether embargoed content reached the outside world through channels the Judge could not see, and if so, when and by whom.

Element Value
Rounds in the timeline 23, from 17 May to 5 June
Agents 7 (Legal, Platform Trust, PR, Social Media, PR Intern, Intern, Judge)
Channels 6, from monitored internal to unmonitored public
Messages 912
Embargo lift 6:00 PM on 5 June, the final round

Source data

The data arrives as a nested JSON file holding 23 timeline rounds. Each round records an event headline, a short narrative and the messages exchanged in that round. The seven agents communicate over six channels of decreasing accountability, and the Judge was assigned to the communications huddle partway through the timeline as the compliance monitor.

Flattening to three tables

The nested timeline was flattened into three working tables. A message table of 912 rows holds one row per message with its sender, channel, recipients, content and the agent’s three private reasoning fields named reacting, rationalizing and deliberating. A round table of 23 rows holds each round’s headline and narrative. A network table holds the directed links between agents derived from the recipient lists.

Types were fixed during the read so that every downstream axis orders correctly and every logical flag stays logical. Agent identity and channel were read as factors with explicit level orders, round and word counts as integers, and the embargo and recipient flags as logicals.

Reading the cleaned message table with fixed column types 
library(tidyverse)

messages_tbl <- readr::read_csv(
  "data/messages_clean.csv",
  col_types = readr::cols(
    round_idx        = readr::col_integer(),
    round_hour_dt    = readr::col_datetime(),
    agent_id         = readr::col_factor(levels = c(

    recipients_is_all     = readr::col_logical(),
    embargo_hit           = readr::col_logical(),
    .default              = readr::col_character()
  )
)

Data dictionary

The message table is the working unit of the analysis. Its main fields are below. Each row is one message.

Field Type Meaning
round_idx integer Round number, 1 to 23
round_hour_dt datetime In-scenario hour the round represents
message_id character Unique identifier for the message
agent_id factor The agent that sent the message
channel factor Channel the message was sent on, from the accountability ladder
message_type character Broadcast or reply
responding_to character The message_id this message replies to, if any
recipients_csv character Recipient short names separated by semicolons, or ALL for a broadcast
recipients_is_all logical Whether the message went to everyone
content character The visible message text
reacting character The agent’s private reaction to the moment
rationalizing character The agent’s private justification of its choice
deliberating character The agent’s private weighing of options
word_count integer Words in content
embargo_keyword_count integer Count of embargo-related keywords found in content
embargo_hit logical Whether the message carried embargo content

The round table holds one row per round with its event_headline and event_narrative, and the network table holds the directed from and to agent links with a message-count weight. Several analysis fields, such as the calm-period channel share and a deviance flag, were added to the message table during the computation stage and are described under Analytical Computation.

The channel accountability ladder

The central reference structure is the channel accountability ladder. Every channel was ranked from most to least accountable and marked for whether the Judge can see it. A legitimate announcement should travel down this ladder, starting as internal discussion on a monitored channel and ending as an attributed public post. Content that reaches an unmonitored public channel without that accountable path is the signal the rest of the analysis looks for.

Building and drawing the accountability ladder 
library(tidyverse)

ladder <- tibble(
  channel   = c("Communications huddle","One-on-one chat","Side huddle",
                "Official post","Personal post","Anonymous post"),
  rank      = 1:6,
  layer     = c("Internal","Internal","Internal","Public","Public","Public"),
  monitored = c(TRUE, TRUE, FALSE, FALSE, FALSE, FALSE)
)

ggplot(ladder, aes(x = 7 - rank, y = reorder(channel, 7 - rank), fill = monitored)) +
  geom_col(width = 0.7) +
  geom_text(aes(label = layer), hjust = 1.1, colour = "white", size = 3.2) +
  scale_fill_manual(values = c(`TRUE` = "#2F855A", `FALSE` = "#9B2C2C"),
                    labels = c(`TRUE` = "Monitored by the Judge", `FALSE` = "Unmonitored"),
                    name = NULL) +
  labs(x = "Accountability (longer bar means more visible to the Judge)",
       y = NULL, title = "The six channels, ranked by accountability") +
  theme_minimal(base_size = 12) +
  theme(axis.text.x = element_blank(), panel.grid = element_blank(),
        legend.position = "top")

Other reference structures

Two further structures were prepared. A recipient lookup maps the short recipient names such as legal or pr to the full agent identities such as legal_agent or pr_agent, so that network edges connect to real nodes rather than dropping out. A topic vocabulary gives each of six topics, covering legal, product trust, merger, media, market and compliance language, a keyword pattern so that every message can be tagged by the topics its content matches.

channel_palette <- c(
